Command line application development in Qt

The first application you are going to write in Qt is the same as the first program you wrote in Java which is, Hello World! This is to ensure that all the libraries are working as expected and that the settings are correct for building multi-class applications.

Set up a project in Qt

Before you get started developing your first C++ application, lets start by setting up a project in QtCreator. First open QtCreator which can be found in the start menu.

Once QtCreator is open you will be greeted by the getting started pane.

This includes links to a number of useful tutorials and examples which may help you become familiar with the built in Qt libraries, particular for user interface development. The tutorials may be useful when working on your assignment although it would be better to complete next week's lab before attempting these tutorials in your own time.

For now lets focus on developing a simple console application in C++. To do this first create a new project by clicking the rather oversized new project button in the centre, through the file menu, or by using the shortcut Ctrl-N.


Once you have selected a new project you will need to select what type of project you will be constructing. It is important to select the appropriate style of project at this point of development as each project type includes a specialised project configuration file depending on the settings selected.

For today you will be creating a Qt Console application project which can be seen selected in the Figure to the right.

When you run your application this time you should find that it prints Hello World! and that you can press enter to terminate the program.

Info

As you continue to write C++ programs you may find that you don't want to write std:: in front of every method you call from the STL; such as std::cout, std::cin or std::endl. To avoid this you can write using namespace std; at the top of you implementation file. Make sure that you never do this in your header files!

Inheritance in C++

Now that you know how to create a project in Qt you're ready to move on to a multi-class application. To do this first create a second C++ Console Application in Qt Creator. This project should be called 'Vehicles'. In this project you are going to implement the inheritance structure seen below.



Each of these objects will be written as a class.

Info

Unlike Java, C++ does not have an interface or abstract keyword; rather you can create abstract classes and interfaces using pure virtual methods.

Use the follow skeleton code to help with this exercise. You will need to create the following files to complete this task. Vehicle.h, MotorVehicle.h, Bus.h, Bus.cpp, Car.h, Car.cpp, Bicycle.h, Bicycle.cpp

Building the Abstract Classes

First you will need to create the abstract classes. For these classes you only need header files. Only the concrete classes will need implementation (.cpp) files.

The Vehicle class should include the following methods

#include <string>

namespace week02 {
	class Vehicle {
	public:
		Vehicle(int numberOfPassengers, 
			int topSpeed, 
			int numberOfWheels, 
			std::string color = "red");

		virtual ~Vehicle();

		virtual std::string getColor();

		virtual int getTopSpeed();

		virtual int getNumberOfWheels();

		virtual int getNumberOfPassengers();

		virtual int getSafetyRating() = 0;

	protected:
		int m_numberOfPassengers;
		int m_topSpeed;
		int m_numberOfWheels;
		std::string m_color;
	};	
}

Note

A method that is marked virtual and equal to 0 is called a pure virtual method, such as the getSafetyRating() in the Vehicle class. This type of method behaves in a similar way to a method marked abstract in Java.

The MotorVehicle class should include the following methods and should publicly inherit from Vehicle 

#include "Vehicle.h"

namespace week02 {
	class MotorVehicle : public Vehicle {
	public:
		MotorVehicle(int numberOfPassengers,
			int topSpeed, 
			int numberOfWheels, 
			double kilometresPerLitre);

		MotorVehicle(int numberOfPassengers,
			int topSpeed, 
			int numberOfWheels, 
			std::string color, 
			double kilometresPerLitre);

		virtual ~MotorVehicle();

		virtual double getKilometresPerLitre();

	protected:
		double m_kmpl;
	};
}

Note

In C++ we have three types of inheritance. Public, Protected and Private inheritance. Each of these will cause the inheritance behaviour to change. So be careful which one you choose.

Although you have been provided the scaffold for the following two abstract classes all but the getSafetyRating() methods need to be implemented. This therefore means that you need to write the appropriate constructors, destructors and getter methods for the member variables of each class. Each constructor should be implemented using an initialisation list. If you are unfamiliar with this syntax it is as follows.

Constructor(int variable1, int variable2) 
    : m_variable1(variable1)
    , m_variable2(variable2)

The destructors for each class should be empty methods and the getters should simply return the member variables which they match to.

Building the Concrete Classes

There are a couple of things which are important to remember when implementing the three concrete classes in this exercise. All three should be in the week02 namespace. As you are required to create both a header (.h) and implementation (.cpp) file for each of these classes, ensure that your implementation includes the using namespace week02; statement in each file.

The Car class

The first class you should build is the Car class. To do this add a class in Qt Creator which will create both the header (.h) and implementation (.cpp) files. The car class should publically inherit the MotorVehicle class and should have two new variables m_numberOfAirBags and m_abs which indicate the number of airbags in a car and whether the car as an automatic braking system respectively. The Car class constructor should match the method signatures seen below after incorporating these two additional variables. 

Car(int numberOfPassengers, 
	int topSpeed, 
	double kilometresPerLitre, 
	int numberOfAirBags = 2, 
	bool abs = true, 
	int numberOfWheels = 4);

Car(int numberOfPassengers, 
	int topSpeed, 
	double kilometresPerLitre, 
	std::string color, 
	int numberOfAirBags = 2, 
	bool abs = true, 
	int numberOfWheels = 4);

Once you have set up your constructors you now need to implement the getSafetyRating() method which should match the following.

int Car::getSafetyRating() {
    int safetyRating = 0;
    if (m_numberOfAirBags >= 4) {
        safetyRating += 3;
    } else if (m_numberOfAirBags >= 2) {
        safetyRating += 2;
    } else if (m_numberOfAirBags > 0) {
        safetyRating += 1;
    }

    if (m_abs) {
        safetyRating += 2;
    }
    return safetyRating;
}

The Bus class

The next class to implement is the Bus Class. Follow the same steps used to build the Car class except that the two additional variables required for a bus are m_seatBeltsInstalled and m_standingPassengersAllowed which indicate whether a bus has seat belts installed and whether the bus allows for standing passengers respectively. These two additional variables can be used to implement the following two constructors for the bus class.

 Bus(int numberOfPassengers, 
	int topSpeed, 
	double kilometresPerLitre, 
	bool seatBeltsInstalled = false, 
	bool standingPassengersAllowed = true, 
	int numberOfWheels = 6);
 
 Bus(int numberOfPassengers, 
	int topSpeed, 
	double kilometresPerLitre, 
	std::string color, 
	bool seatBeltsInstalled = false, 
	bool standingPassengersAllowed = true, 
	int numberOfWheels = 6);

Once you have set up your constructors you now need to implement the getSafetyRating() method which should match the following.

int Bus::getSafetyRating() {
    int safetyRating = 0;

    if (m_seatBeltsInstalled) {
        safetyRating += 3;
    }
    if (!m_standingPassengersAllowed) {
        safetyRating += 2;
    }
    return safetyRating;
}

The Bicycle class

the last class to implement is the Bicycle Class. Unlike the previous two vehicles a Bicycle is not a motor vehicle so therefore it should only inherit from the Vehicle class. The Bicycle class has a single additional variable which is the m_helmetUsed to indicate if the rider of the bike is using a helmet. Similar to the previous two classes Bicycle has two constructors which have the following method signatures.

Bicycle(int numberOfPassengers, 
	int topSpeed, 
	bool helmetUsed = false, 
	int numberOfWheels = 2);

Bicycle(int numberOfPassengers, 
	int topSpeed, 
	std::string color, 
	bool helmetUsed = false, 
	int numberOfWheels = 2);

Once you have set up your constructors you now need to implement the getSafetyRating() method which should match the following.

int Bicycle::getSafetyRating() {
    if (m_helmetUsed) {
        return 5;
    } else {
        return 0;
    }
}

Testing it all works

Once you have implemented the three concrete classes along with the two abstract classes replace the code in the main.cpp of the Qt Creator application with the following code.

#include <iostream>

#include "Bicycle.h"
#include "Bus.h"
#include "Car.h"

using namespace week02;

int main(int argc, char* argv[]) {

	Bicycle bicycle1(1, 15);
	Bicycle bicycle2(1, 10, std::string("Blue"));
	Bicycle bicycle3(1, 15, true);
	Bicycle bicycle4(1, 10, std::string("Yellow"), true);
	Bicycle bicycle5(1, 15, true, 3);
	Bicycle bicycle6(1, 10, std::string("Green"), true, 3);

	std::cout << bicycle1.getSafetyRating() << std::endl;
	std::cout << bicycle2.getSafetyRating() << std::endl;
	std::cout << bicycle3.getSafetyRating() << std::endl;
	std::cout << bicycle4.getSafetyRating() << std::endl;
	std::cout << bicycle5.getSafetyRating() << std::endl;
	std::cout << bicycle6.getSafetyRating() << std::endl;

	Bus bus1(1, 75, 10, true, false);
	Bus bus2(1, 75, 10, std::string("Blue"), true, true);
	Bus bus3(1, 75, 10);
	Bus bus4(1, 75, 10, false, true);

	std::cout << bus1.getSafetyRating() << std::endl;
	std::cout << bus2.getSafetyRating() << std::endl;
	std::cout << bus3.getSafetyRating() << std::endl;
	std::cout << bus4.getSafetyRating() << std::endl;

	Car car1(1, 75, 10, 4);
	Car car2(1, 75, 10, std::string("Blue"));
	Car car3(1, 75, 10);
	Car car4(1, 75, 10, 6, true, 4);
	Car car5(1, 75, 10, std::string("Green"), 6, true, 4);
    
	std::cout << car1.getSafetyRating() << std::endl;
	std::cout << car2.getSafetyRating() << std::endl;
	std::cout << car3.getSafetyRating() << std::endl;
	std::cout << car4.getSafetyRating() << std::endl;
	std::cout << car5.getSafetyRating() << std::endl;

	return 0;
}
