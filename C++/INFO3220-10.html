Why C++ Templates are not the same as the Template Design Pattern

The Standard Template library which you were introduced in week 8's tutorial is designed such that each container, iterator and algorithm is templated. A template in C++ is similar to a generic in Java where the object created does not explicitly define which data type is used. As a result all containers in the STL can be used with many different data types rather than needing to be rewritten for each one.

The ability to created templated classes is an important property of C++ and over the next few weeks you will be writing a number of templated data structures such as a Templated StackQueue this week and a Templated Shared Pointer in Week 12.

The template design pattern, unlike C++ templates, is a behavioural design pattern that defines a high level set of algorithms where the complex steps are deferred to subclasses. By designing an algorithm library in this fashion you can modify particular steps of an algorithm's structure without modifying the algorithms structure. Further, by designing an algorithm library in this fashion methods can be designed to handle drastically different data structures using a common library. For example, the find methods on a set will be performed in a different way to a vector.

C++ templates and the template design patterns do overlap. The STL offers a library of algorithms which are implemented using the template design pattern. The algorithms library offers a number of methods which can be applied to STL containers such as find, max, min, shuffle and sort. A full list of the methods offered by the algorithms library can be found here.

Creating a Templated Stack

Your first task this week is to update your StringStackQueue from week 8's tutorial and turn it into a StackQueue<T>.

This will involve updating the interfaces provided in week 8 for a StringStack and StringQueue to use the Generic Stack and Queue as shown below.

namespace Week10
{
    template <class T>
    class Stack
    {
    public:
        virtual ~Stack(){}

        /**
         * @brief empty checks if the stack is empty
         * @return true if empty otherwise false
         */
        virtual bool empty() const = 0;

        /**
         * @brief size checks the number of items in the stack
         * @return the number of items in the stack
         */
        virtual size_t size() const = 0;

        /**
         * @brief peek is the item at the top of the stack
         * this method should not alter the state of the stack
         * @return a reference to the top of the stack
         */
        virtual const T& peek() const = 0;

        /**
         * @brief push adds an new item to the top of the stack
         * @param item the new item to add to the top of the stack
         */
        virtual void push(const T &item) = 0;

        /**
         * @brief pop removes the item on the top of the stack
         * if the stack is empty return a default constructed T
         * @return the item that was previously on the top of the stack
         */
        virtual T pop() = 0;

        /**
         * @brief operator = overwrites the current stack with a new stack
         * @param stack the new stack
         * @return a reference to the current stack
         */
        virtual Stack<T>& operator=(const Stack<T> &stack) = 0;
    };
}

namespace Week10
{
    template <class T>
    class Queue
    {
    public:
        virtual ~Queue(){}

        /**
         * @brief empty checks if the queue is empty
         * @return true if empty otherwise false
         */
        virtual bool empty() const = 0;

        /**
         * @brief size checks the number of items in the queue
         * @return the number of items in the queue
         */
        virtual size_t size() const = 0;

        /**
         * @brief front is the item at the front of the queue
         * this method should not alter the state of the queue
         * @return a reference to the first item in the queue
         */
        virtual const T& front() const = 0;

        /**
         * @brief enqueue adds a item to the end of the queue
         * @param item the item to add at the end of the queue
         */
        virtual void enqueue(const T &item) = 0;

        /**
         * @brief deqeue deletes an item from the front of the queue
         * if the deqeue is empty return a default constructed T
         * @return the first item from the queue before deleted
         */
        virtual T deqeue() = 0;

        /**
         * @brief operator = overwrites the current queue with a new queue
         * @param queue the new queue
         * @return a reference to the current queue
         */
        virtual Queue<T>& operator=(const Queue<T> &queue) = 0;
    };
}
